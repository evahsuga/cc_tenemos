# 業務自動化ランチャー 開発基本計画書

**プロジェクト名**: 業務自動化ランチャー（仮称）  
**作成日**: 2024年12月29日  
**バージョン**: 1.0  

---

## 📋 プロジェクト概要

### 目的
弥生販売とカラーミーショップの操作を、ボタン1クリックで自動実行し、
業務効率を劇的に向上させる専用ランチャーアプリケーションを開発する。

### 背景
- 現状：同じ操作を毎日何度も手作業で実行している
- 課題：RPAソフトは遅い（12秒）、不安定、高コスト（年間30万円〜）
- 目標：「てるの助」のような高速性（1秒以内）を実現

### 期待効果
- ⏱️ 作業時間：現行の1/15に短縮（12秒 → 0.8秒）
- 💰 コスト削減：RPAライセンス費用0円
- 🎯 安定性向上：画面変更に強い
- 🚀 拡張性：今後の追加要件にも柔軟に対応

### 技術スタック

#### フロントエンド
- **Electron**: デスクトップアプリケーション基盤
- **HTML/CSS/JavaScript**: UI実装

#### 自動化エンジン
- **Puppeteer**: Web自動化（カラーミーショップ）
- **pywinauto**: Windows自動化（弥生販売）
- **Node.js**: メインロジック

#### データベース
- **SQLite**: ログ・設定の保存（オプション）

## 📐 機能設計

### 1. メイン機能

#### 1-1. カラーミーショップ自動操作

**シナリオ例**: 受注処理
```
工程1: 管理画面にログイン
工程2: 受注一覧を開く
工程3: 受注IDで検索
工程4: 受注詳細を開く

実行時間目標: 1秒以内
```

**実装方法**: Puppeteer

#### 1-2. 弥生販売自動操作

**シナリオ例**: 顧客情報表示
```
工程1: 顧客検索画面を開く（Ctrl+F）
工程2: 顧客コードを入力
工程3: 検索実行
工程4: 詳細画面を表示

実行時間目標: 0.5秒以内
```

**実装方法**: pywinauto

### 2. サポート機能

#### 2-1. ログ機能
- 実行履歴の記録
- エラーログの保存
- 実行時間の計測

#### 2-2. 設定機能
- ログイン情報の保存（暗号化）
- デフォルト値の設定
- ショートカットキーの設定

#### 2-3. エラーハンドリング
- リトライ機能（最大3回）
- エラー時の通知
- スクリーンショット保存

## ⚠️ リスク管理

### 技術的リスク

| リスク | 影響度 | 対策 |
|--------|-------|------|
| Puppeteerが動作しない | 高 | Selenium等の代替技術を用意 |
| 弥生販売の要素が取得できない | 中 | 画像認識の併用を検討 |
| 実行速度が目標に届かない | 低 | 処理の最適化 |

### 運用リスク

| リスク | 影響度 | 対策 |
|--------|-------|------|
| カラーミーの画面変更 | 中 | 柔軟な要素指定方法を採用 |
| 弥生販売のバージョンアップ | 中 | バージョン別の処理分岐 |
| ログイン情報の管理 | 高 | 暗号化して安全に保存 |
